Simplified Git and GitHub
Recently, I’ve started exploring Git and GitHub for version control. Using Git’s modern workflow has allowed me to move away from outdated methods. Here’s a step-by-step guide on how to set up SSH and use Git effectively:

  1. Generate SSH Key for Git
    Open Terminal
    Navigate to your desired file path or work from your home directory.

Generate RSA Key Pair
Use the following commands to create an SSH key pair:

Alternatively, you can use:

ssh-keygen -t rsa
Find the RSA Key
Your public and private keys will be stored in the ~/.ssh directory by default.

Check the Key
Use cat to display the contents of your public key:

  1. Add SSH Key to GitHub
    Open a Repository on GitHub
    Create or navigate to an existing repository.
    Ensure you configure a .gitignore file to exclude unnecessary files from your version control.

Add SSH Key to GitHub
Go to Settings → Access → SSH and GPG keys and follow these steps:

Title: Enter a descriptive name (e.g., your email or device name).
Key: Paste the public key (starts with ssh-rsa).
After successful deployment, you will see your SSH key listed under “Authentication keys.”

  1. Clone a Repository via SSH
    Find the SSH Path
    Navigate to your GitHub repository and click Code → SSH. You will see a path similar to:

Clone the Repository
Open a terminal, navigate to the folder where you want to store the repository, and run:

git clone git@github.com:yourgithubname/repository.git

    Open SourceTree, a graphical interface for Git.
    Pull changes from the repository using the Pull command.
    Stage your changes and upload them to GitHub via Push.
