FSM

2024-05-29

Here is a brief intro to FSM in GODOT 4

The script: class of “state”:
It contains basic var that a state should acquire.

  1. what to do by frame
  2. what to do when enter or leave
  3. the signal when it leave

Based on class “state”, we can develop a further state: “EnemyIdle”

  1. specify the char that affect by FSM
  2. overwrite the funcs including enter(), update(), physics_update()

build a controller for FSM

  1. go through all child node, add it into the dictionary if child node is state class
    example: states {run: String: run: Node, idle: String: idle: Node …}
  2. the state signal connect to on_child_transition once it registered
  3. urge corresponding state work (notice! the update func of state is not a build-in func which means it is drive by this FSM controlloer)
  4. specify the current_state
  5. manage the transition between different state
