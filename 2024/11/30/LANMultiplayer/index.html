Import 3D model to godot 4.2

一般而言模型有集中格式

  • fbx - Godot 一般支持。包含材质、贴图、灯光和摄像机数据。文件较大
  • gltf/glb - Godot 全面支持。加载速度快，包括动画、材质和骨骼。
  • obj - Godot 一般支持。传统的 3D 模型格式，常用于静态模型。

Animation
拖拽进godot后在import 的 advance 里可以更加详细地看清楚细节
并且在Root type中可以根据需求改变node（比如把角色模型的root type从node 3D改为charactor body3D）
同时也需要设定各种动画的loop mode，有些动画需要循环的诸如running和idle

Collision
我们不需要一个个在godot里为物品创建collision shape，接上查看细节的步骤。
选中mesh，选中Generate-Physics，在shape type中就能改变。对于大多数模型，simple convex就行了。

Child Node
一个复杂的模型可能包括很多不同的部分。右击GLB文件中选择 New inherited scene 就可以把这个复杂的模型以node的形式拆解。

为mesh自动生成collisionshape
选中mesh node，在主视窗上部选中mesh，增加create trimesh static body

改变颜色
方法 1：通过 Material Override 修改颜色
选中你的 MeshInstance3D 节点。
在 Inspector 面板 中找到 Material Override 属性。
点击右侧的 + 按钮，选择 StandardMaterial3D。
在新的材质设置中：
找到 Albedo（基色）部分。
修改 Color 为白色。

Input.MouseModeEnum.Visible

默认模式，鼠标指针可见并能自由移动。
鼠标可以移出窗口，不会影响应用程序。
常用于 UI 操作和非全屏应用。
Input.MouseModeEnum.Hidden

鼠标指针隐藏，但仍然可以自由移动。
常用于需要鼠标功能但不希望显示光标的场景。
Input.MouseModeEnum.Captured

鼠标指针被捕获（锁定在窗口内），并隐藏。
鼠标移动会产生相对坐标（InputEventMouseMotion.Relative），即鼠标移动的增量。
用途：第一人称射击、第三人称视角旋转、飞行模拟器等。
Input.MouseModeEnum.Confined

鼠标指针可见，但限制在窗口范围内，不能移出窗口。
常用于策略游戏或其他需要保持鼠标在窗口范围内的场景。
