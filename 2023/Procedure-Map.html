Tips - Debug

When you instantiate one class like below

如果 “Walker.new” 中的 “W” 使用小写字母，它将引用变量 “walker”，而不是之前创建的类 “Walker”。
If “W” in “Walker.new” is in lowercase, it will refer to the variable “walker” instead of the class “Walker,” which was created earlier.

为了解决这个问题，你可以更改类中的 class_name，或者更改变量名称。

Tips - Why Use ceil()

小贴士 - 为什么使用 ceil()

在这里，我们提供一个示例，说明为什么在房间的长度和宽度为奇数时，需要使用 ceil() 函数来确定左上角的位置。
Here I provide a sample that explains why we use the ceil() function to locate the top-left corner when the length and width are odd numbers.

示例 Example

  • position (3, 2): 房间中心 (room center)
  • top_left (0, 0): 计算出的左上角 (calculated top-left)
  • size (5, 3): 长 × 宽 (length × width)
  • 1/2 size (2.5, 1.5): 一半的大小 (half size)

说明 Note

ceil() 函数确保在计算 top_left_corner 时，将浮点数向上取整，避免坐标偏移导致房间位置错误。
The ceil() function ensures that when calculating the top_left_corner, the floating-point numbers are rounded up to prevent coordinate offsets that could result in incorrect room placement.

注意 (Note):
if borders.has_point(room_floor_step): 这一行代码确保房间地板步骤位于边界内部。
The if borders.has_point(room_floor_step): condition ensures that the room floor steps fall inside the border.

Player Initialization

玩家初始化

在本节中，我们将为这个迷宫/地牢（或者随你怎么称呼）创建一个角色（avatar）。
Besides, we will create an avatar for this maze/dungeon or whatever you call it.

此外，还会在迷宫/地牢中生成一个最远端的出口。
Besides, there will be a self-generated exit port located farthest from the maze/dungeon.

目标 Objectives

  1. 角色创建: 让玩家能够在迷宫中自由移动。
    Avatar creation: Allow the player to move freely in the maze.

  2. 自动生成出口: 出口将根据迷宫布局自动生成在最远的位置。
    Auto-generated exit: The exit will be automatically placed at the farthest point in the maze.

  3. 关卡切换: 当角色到达出口时，将进入下一关。
    Level transition: When the avatar enters the exit, it will proceed to the next level.
